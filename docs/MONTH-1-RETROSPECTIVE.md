# Month 1 Post-Launch Retrospective Framework

**Owner:** Chief of Staff
**Timing:** Day 31 post-launch
**Duration:** 90 minutes
**Participants:** All agents + founding team

---

## Pre-Retro Checklist (Day 28–30)

Before the retrospective, the following data must be pulled and pre-populated:

| Data Point | Source | Owner |
|------------|--------|-------|
| Units sold by channel | Shopify Admin → Analytics → Sales by channel | CMO / Growth |
| Revenue & gross margin | Shopify Admin → Finances → Profit report | CFO / Ops |
| CAC by channel | Google Ads + Meta Ads Manager + Shopify attribution | CMO / Growth |
| Waitlist → buyer conversion | Shopify customer tags cross-referenced with waitlist CSV | CMO / Growth |
| Site visitor → buyer conversion | Google Analytics 4 → Conversions → Purchase event | CMO / Growth |
| Top-performing ad creative | Google Ads + Meta Ads Manager → sort by ROAS | CMO / Growth |
| NPS score (Day 7 + Day 30) | Google Forms / Typeform responses (see NPS-SURVEY.md) | Customer Success |
| Tutorial completion rates | Tutorial site analytics → project completion events | Product |
| Top support ticket categories | Gmail / Helpscout tags (see SUPPORT-INBOX-TAGGING.md) | Customer Success |
| Defect / missing component rate | Feedback Tracker → filter Category = `hardware` or `bug` (see FEEDBACK-TRACKER.md) | Customer Success |
| Fulfilment accuracy & on-time % | Shopify Shipping → fulfilment reports + carrier tracking | Operations |
| Inventory remaining | Shopify Inventory report → weeks of cover at current run rate | Operations |
| Supplier lead time performance | Purchase orders vs. actual delivery dates (internal spreadsheet) | Operations |

## Output

After the retrospective, the Chief of Staff produces:

1. **Month 1 Scorecard** — one-page summary of all metrics vs. targets (shared in project docs)
2. **Month 2 Priority List** — ranked list of actions from Part 5, with owners and deadlines
3. **Issue backlog updates** — new issues created in Paperclip for each Month 2 action item

## Scheduling

- **Retrospective date:** Day 31 post-launch

- **Follow-up actions due:** Day 33 (owners assigned, issues created)
